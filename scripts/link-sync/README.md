# link-sync — YouTube affiliate-link sync & staleness watch

Keeps the affiliate/referral links in **every** video description on your
channel in sync with a single canonical registry, and flags links that have
gone stale or dead — including the Amex case, where a referral link expires to
a "no longer available" page that still returns HTTP 200.

## How it works

`registry.ts` is the source of truth: one entry per program with its
**current** link, its **known prior** links (`aliases`), and how to recognize
its links in a description. A scan pulls every video on your channel and sorts
each link it finds into:

| Status | Meaning | Auto-fixed? |
|---|---|---|
| ✅ current | already the canonical link | — |
| 🟡 stale | equals a known old link (`alias`) | **yes** (→ current) |
| 🔵 review | same domain, unrecognized URL (could be a different product) | no — needs your eyes |
| 🔴 dead | destination is expired/invalid (live check) | no |
| 🟠 needs-current | matched a program with no current link set yet (Amex after a rotation) | no |
| ⚪ orphan | affiliate-shaped link not in the registry (forgotten one-off) | no |

**Only `stale` links are ever rewritten automatically**, and only to a URL you
explicitly put in the registry. Everything else is reported, never touched.

### The Amex loop
Amex links can't be regenerated by a script — only from your account. So when
one dies: the scan flags it 🔴/🟠 → you grab a fresh link from Amex → paste it
as the program's `currentUrl` (move the dead one into `aliases`) → `apply`
propagates it to every affected video.

## One-time setup

1. **Google Cloud project** → enable **YouTube Data API v3**
   (console.cloud.google.com → APIs & Services → Library).
2. **OAuth consent screen**: External, add yourself as a test user (so it works
   without verification). Scope: `youtube.force-ssl`.
3. **Credentials → Create OAuth client ID → Desktop app**. Copy the client ID
   and secret.
4. Add to `.env.local` in the repo root:
   ```
   YT_CLIENT_ID=xxxx.apps.googleusercontent.com
   YT_CLIENT_SECRET=xxxx
   ```
5. Mint a refresh token (opens a browser for one consent click):
   ```
   npm run linksync:auth
   ```
   Paste the printed `YT_REFRESH_TOKEN=…` line into `.env.local`.

## Usage

```bash
npm run linksync:scan                 # scan + live check → output/report.md
npm run linksync:scan -- --no-live    # registry diff only, no network fetches
npm run linksync:apply                # DRY RUN: print every rewrite it would make
npm run linksync:apply -- --commit    # push the safe rewrites
npm run linksync:apply -- --commit --limit=25   # quota guard (~50 units/update)
npm run linksync:apply -- --only=VIDEOID
```

Reports land in `scripts/link-sync/output/`.

## Quota

Scanning the whole channel is cheap (~1 unit per 50 videos). Each description
update costs ~50 units; the default 10,000 units/day ≈ **200 edits/day**. Use
`--limit` to stay under it on big back-catalog sweeps.

## Maintaining the registry

- Rotated a link? Move the old URL into `aliases`, set the new `currentUrl`.
- New program you promote on YouTube? Add an entry with `domainMatch` so the
  scanner stops calling it an orphan.
- Same domain, multiple products (Chase business vs personal)? Use
  `contextKeywords` so a link is only matched when the description text nearby
  names the right product.
